# BINGO MAX - Deployment Guide

## Quick Deploy (FREE)

### 1. Push to GitHub
```bash
git init
git add .
git commit -m "BINGO MAX - Multiplayer Bingo Game"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/BINGO-MAX.git
git push -u origin main
```

### 2. Deploy Frontend to Vercel (FREE)
1. Go to https://vercel.com
2. Click "New Project" â†’ "Import Git Repository"
3. Select your BINGO-MAX repository
4. Click "Deploy"
5. Copy your Vercel URL: `https://bingo-max.vercel.app`

### 3. Deploy Backend to Railway (FREE tier $5 credit)
1. Go to https://railway.app
2. Click "New Project" â†’ "Deploy from GitHub"
3. Select your BINGO-MAX repository
4. Add environment variable:
   - Key: `DATABASE_URL`
   - Value: Railway auto-creates this when you add PostgreSQL
5. Add PostgreSQL plugin:
   - Click "Add" â†’ "Add from marketplace" â†’ "PostgreSQL"
   - Railway will auto-populate `DATABASE_URL`
6. Deploy
7. Copy your Railway URL from Deployments tab (e.g., `https://bingo-backend-xyz.railway.app`)

### 4. Connect Frontend to Backend
After backend is deployed on Railway:

**In Vercel:**
1. Go to your project settings
2. Add Environment Variable:
   - Key: `VITE_API_URL`
   - Value: `https://your-railway-backend.railway.app` (your Railway URL)
3. Redeploy

**Update Frontend Code:**
In `client/src/lib/game-state.tsx`, change API calls to use the environment variable.

---

## Environment Variables

### Railway Backend (.env)
```
DATABASE_URL=<auto-generated by Railway>
NODE_ENV=production
PORT=5000
```

### Vercel Frontend (.env)
```
VITE_API_URL=https://your-railway-backend.railway.app
```
